# 项目核心踩坑与排查记录

---

## 踩坑 1：训练/推理 prompt 不一致导致模型退化

### 现象

LoRA 微调后，模型输出变得模板化、模糊，大量出现"未知"、"未显示"。打分从 Baseline 的 14.5 分降到 10.75 分。

### 排查

对比训练数据和推理 prompt，发现完全不一致：

```
训练数据：
  用户问：请描述这张驾驶场景图片中的车辆、行人和关键物体。
  模型答：3个小汽车,1个卡车,2个行人

推理时实际问的是：
  请分析这张驾驶场景图片，输出以下信息：
  1. 车道线数量和类型  2. 前方车辆位置  3. 交通标志  4. 驾驶风险
```

### 根因

同样的 500 张图片，问答格式不同 → Loss 下降方向不同 → 最终权重完全不同。Loss 从 66.80 降到 30.55，但模型学到的是回答旧 prompt 的能力。**Loss 下降 ≠ 输出质量变好。**

### 解决方案

改训练数据 prompt 为四项格式，和推理完全一致。

## 踩坑 2：量化配置不一致导致推理效果打折

### 现象

训练 Loss 正常下降（64→29），但推理输出质量一般，LoRA 收益没有完全体现。

### 排查

逐行对比训练和推理脚本的量化配置：

| 配置项 | 训练 | 推理 | 一致？ |
|--------|------|------|--------|
| `bnb_4bit_compute_dtype` | `bfloat16` | `float16` | ❌ |
| `bnb_4bit_quant_type` | `"nf4"` | 默认 `"fp4"` | ❌ |
| `torch_dtype` | `bfloat16` | `float16` | ❌ |

### 根因

QLoRA 的 LoRA 修正量是针对特定量化配置学的。推理时配置变了，基础权重的数值表示就变了，修正量部分失配。**这个 bug 最隐蔽：不会报错，不会崩溃，只是静默降低效果。**

### 解决方案

推理脚本量化配置改成和训练完全一致（bf16 + nf4）。

## 踩坑 3：模板句导致模型背模板

### 现象

修复前两个问题后重跑评估，车道线和风险项仍然每张图输出固定文本：
- "根据图片判断车道线数量和类型。" — 500条全一样
- "根据场景判断潜在风险。" — 500条全一样

### 根因

nuScenes 只有物体标注，没有车道线和风险标注。04 脚本对缺失项填了固定模板句。模型把模板当正确答案学了，不看图直接输出。

### 解决方案

用 Baseline 模型对 500 张训练图推理，提取 pseudo-label 补全缺失标注。虽然 pseudo-label 不完美（车道线约 60% 准确），但每张图内容不同，模型能学到"看图描述"的正确行为。

## 踩坑 4：推理重复循环问题

### 现象

20% 的图模型陷入列举循环，同一句话重复 30-40 遍：
```
- 一辆银色轿车停在左侧路边。- 一辆银色轿车停在左侧路边。- 一辆银色轿车停在左侧路边。
```

LLM 生成时的经典问题——退化输出（degenerate output）。模型在生成到某个 token 时概率分布变得很尖，导致反复选同一个 token。4bit 量化模型 + greedy decoding + 没有重复惩罚 → 容易循环。

### 调参过程

| 方案 | penalty | max_tokens | no_repeat_ngram | 结果 |
|------|---------|-----------|-----------------|------|
| 无 penalty | 无 | 512 | 无 | 16% 循环，最长 974 字 |
| penalty=1.2 | 1.2 | 512 | 无 | 仍有循环，1124 字 |
| penalty=1.5 | 1.5 | 512 | 无 | 更长，换措辞说废话 |
| penalty=1.5+ngram | 1.5 | 512 | 3 | 完全乱码（ASCII art、代码块） |
| **无 penalty+截断** | **无** | **256** | **无** | **正常，20% 轻微循环被截断控制** |

### 关键发现

- `repetition_penalty=1.5` 会把 4bit 模型的概率分布推歪，导致输出乱码
- `no_repeat_ngram_size` 在 4bit 模型上完全不可用，会输出 ASCII art 和英文乱码
- `max_new_tokens=256` 截断是最简单有效的方案，不改变概率分布，只是物理截断

### 最终参数

```python
generated_ids = model.generate(**inputs, max_new_tokens=256)
```

## 后处理：_clean_output 垃圾标记检测

### 背景

即使截断到 256 tokens，模型仍会在末尾生成"聊天废话"（如"希望我的回答对你有所帮助"、"祝您旅途愉快"、emoji 等）。不影响核心分析内容，但影响 Gradio Demo 展示观感。

### 方案

不过滤正常内容，只检测明确的垃圾标记（具体短语 + emoji），从匹配位置截断：

```python
@staticmethod
def _clean_output(text):
    garbage_markers = [
        r'[\U0001F300-\U0001F9FF]',          # emoji
        r'希望我的回答对你有所帮助',
        r'如果您有任何其他问题',
        r'请随时告诉我',
        r'祝您旅途愉快',
        r'oOo',
        r'\\{3,}',
    ]
    cut_pos = len(text)
    for pat in garbage_markers:
        m = re.search(pat, text)
        if m and m.start() < cut_pos:
            cut_pos = m.start()
    if cut_pos < len(text):
        text = text[:cut_pos].rstrip()
        if text and text[-1] not in '。！？…；':
            text += '。'
    return text
```

优点：只切明确垃圾，不会误删正常内容；新垃圾 pattern 可以随时追加。
